# Wiki Query

위키를 검색하고, 관련 페이지를 종합하여 인용 포함 답변을 합성한다.

> **핵심 원칙**: 위키는 이미 교차참조와 종합이 완료된 지식 베이스다. 매번 원본에서 재도출하지 않고, 위키의 축적된 지식에서 답변한다. 좋은 답변은 다시 위키에 저장하여 탐색이 누적되도록 한다.

## 인자 해석

- `$ARGUMENTS`에서 `--save` 제거 → 나머지가 질문
- `--save` 있으면 Phase 4(답변 저장) 실행
- `$ARGUMENTS` 없으면 사용자에게 질문 요청

## Execution Steps

### Phase 0: 위키 존재 확인

1. `.wiki/` 디렉토리 존재 확인
   - 없으면: "위키가 초기화되지 않았습니다. `/wiki-init`를 먼저 실행하세요." 안내 후 중단
2. `.wiki/schema.md` 읽기 — 위키 구조 파악
3. `.wiki/index.md` 읽기 — 전체 페이지 목록 확보

### Phase 1: 페이지 탐색 (Discovery)

질문을 분석하여 관련 페이지를 찾는다:

1. **인덱스 기반 매칭**:
   - 질문의 핵심 키워드를 추출
   - `index.md`의 페이지 제목, 설명, 태그와 매칭
   - 높은 관련성 순으로 후보 페이지 정렬

2. **별칭 매칭**:
   - 질문에 포함된 용어가 페이지의 `aliases`와 매칭되는지 확인
   - 한국어/영어 양방향 매칭

3. **후보 페이지 읽기**:
   - 상위 후보 페이지를 `Read`로 읽기 (최대 10개)
   - 각 페이지의 관련성을 평가

4. **교차참조 체인 탐색**:
   - 읽은 페이지의 `[[wikilinks]]`를 따라 추가 관련 페이지 탐색
   - 1-hop 연결까지 탐색 (A → B에서 B도 읽기)
   - 추가 발견된 관련 페이지도 `Read`로 읽기 (최대 5개 추가)

5. **Grep 보조 검색** (인덱스 매칭이 부족한 경우):
   - `Grep`으로 `.wiki/pages/` 내 키워드 직접 검색
   - 인덱스에서 놓친 페이지 발견

### Phase 2: 답변 합성

발견한 페이지들을 종합하여 답변을 구성한다:

1. **답변 구조**:
   ```markdown
   ## 답변

   [질문에 대한 종합 답변]

   ### 근거

   [답변을 뒷받침하는 위키 페이지 인용]
   - [[page-1]]: [인용 내용 요약]
   - [[page-2]]: [인용 내용 요약]

   ### 참고
   - [[추가 관련 페이지]]
   ```

2. **답변 품질 규칙**:
   - **인용 필수**: 모든 주장에 위키 페이지 참조를 포함
   - **신뢰도 표시**: 여러 페이지가 뒷받침하면 높은 신뢰, 단일 소스면 표시
   - **갭 표시**: 위키에 없는 정보가 필요하면 "위키에 해당 정보가 없습니다. `/wiki-ingest`로 소스를 추가하세요." 안내
   - **모순 표시**: 위키 페이지 간 모순이 있으면 양쪽을 모두 인용하고 모순을 명시

3. **답변 형식 적응**:
   - 비교 질문 → 비교 표 형식
   - "어떻게" 질문 → 단계별 설명
   - "왜" 질문 → 원인-결과 설명
   - 목록 질문 → 글머리 기호 목록

### Phase 3: 답변 출력

합성된 답변을 사용자에게 출력한다.

`--save` 옵션이 **없으면** 여기서 종료.

### Phase 4: 답변 저장 (--save)

`--save` 옵션이 있을 때만 실행:

1. **페이지 생성**:
   - 타입: `analysis`
   - 파일명: 질문을 kebab-case로 변환 (최대 64자)
   - frontmatter: title(질문 요약), type(analysis), references(참조한 위키 페이지 목록, 확장자 제외), tags
   - **주의**: analysis 페이지는 `sources`(원본 소스 파일)가 아닌 `references`(위키 페이지 참조)를 사용
   - 본문: Phase 2에서 합성한 답변을 페이지 포맷에 맞게 정리

2. **인덱스 갱신**:
   - `index.md`의 "분석 (Analysis)" 카테고리에 새 페이지 추가
   - frontmatter 카운트 갱신

3. **교차참조 갱신**:
   - 참조한 기존 페이지의 관련 항목에 새 분석 페이지 역참조 추가

4. **로그 기록**:
   - `log.md`에 query + save 기록 추가

5. **저장 확인**:
   ```
   답변을 [[{page-name}]]으로 저장했습니다.
   ```

## 규칙

- 위키에 없는 정보를 "아는 척" 하지 않음 — 위키 기반 답변만 제공
- 외부 지식이 필요하면 소스 추가를 안내 (`/wiki-ingest`)
- 인용 없는 답변 금지 — 최소 1개 위키 페이지 참조 필수
- `--save`로 저장된 분석 페이지도 위키의 일부로 누적됨
- 언어: 한국어로 출력
