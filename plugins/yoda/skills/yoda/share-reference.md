# Share Mode Reference

yoda share 모드의 전체 실행 파이프라인 및 포맷별 생성 전략 레퍼런스.

---

## 진입점

```
/yoda share <입력> [--format md|web|wiki|slides|pptx] [--audience junior|mid|senior]
```

### 파라미터

| 파라미터 | 필수 | 기본값 | 설명 |
|----------|------|--------|------|
| `<입력>` | 필수 | — | `--from-review`, 파일/디렉토리 경로, 또는 자유 텍스트 |
| `--format` | 필수 | **없음** (사용자에게 물어보기) | 출력 포맷: `md`, `web`, `wiki`, `slides`, `pptx` |
| `--audience` | 선택 | `mid` | 대상 청중 수준: `junior`, `mid`, `senior` |

#### --format 누락 시 동작

`--format`이 지정되지 않으면 다음과 같이 사용자에게 선택을 요청한다:

```
어떤 형식으로 공유할까요?

1. **md** — 마크다운 문서 (docs/ 폴더에 저장)
2. **web** — 인터랙티브 HTML 페이지
3. **wiki** — 위키 페이지 (Confluence 등)
4. **slides** — Marp 프레젠테이션 슬라이드
5. **pptx** — PowerPoint 파일
```

---

## 입력 유형 감지

| 입력 | 감지 방법 | 처리 파이프라인 |
|------|----------|---------------|
| `--from-review` | 플래그 존재 확인 | 직전 review 결과를 대화에서 추출 → 선택 포맷에 맞게 변환 |
| 파일/디렉토리 | `Glob`으로 경로 존재 확인 | 독립 분석 수행 → 콘텐츠 생성 |
| 자유 텍스트 | 위 두 가지에 해당하지 않는 경우 | `Grep`으로 코드베이스 탐색 → 실제 예시 기반 콘텐츠 생성 |

---

## 포맷별 생성 전략

### 1. md (마크다운 문서)

#### 생성 방법

`Write` 도구로 직접 생성한다.

#### 저장 경로

```
docs/yoda/YYYY-MM-DD-[slug].md
```

#### 강화 원칙

| 원칙 | 적용 |
|------|------|
| **Worked Examples** (Sweller) | 완전한 Before/After/Why 예제를 단계별로 분해하여 제시 |
| **이중 코딩** (Mayer) | Mermaid 다이어그램으로 구조/흐름을 시각화 |
| **정교화 질문** (Dunlosky 2013) | 각 섹션 끝에 "생각해볼 점" 질문 배치 |

---

### 2. web (인터랙티브 HTML)

#### 생성 방법

인터랙티브 HTML 페이지를 직접 생성하거나 `frontend-design` 스킬에 위임한다.

#### 저장 경로

```
docs/yoda/YYYY-MM-DD-[slug].html
```

#### 강화 원칙

| 원칙 | 적용 |
|------|------|
| **점진적 공개** (Nielsen) | 접기/탭으로 심화 내용 숨김. 첫 화면은 핵심만 노출 |
| **인출 연습** (Roediger & Karpicke 2006) | 인터랙티브 퀴즈로 독자가 학습 내용을 직접 떠올리게 함 |
| **개인화** (Mayer) | 대화체 사용, 독자의 실제 코드를 예시로 활용 |

---

### 3. wiki (위키 페이지)

#### 생성 방법

마크다운 또는 위키 매크로 형식으로 `Write` 도구로 생성한다. Confluence MCP 서버가 연결되어 있으면 직접 게시할 수 있다.

#### 내러티브 구조

스토리텔링 원칙에 따라 4막 구조로 작성한다:

| 막 | 내용 | 목적 |
|----|------|------|
| **발단** | "우리 팀이 겪은 실제 상황" — 문제가 발생한 맥락 | 공감대 형성, 흥미 유발 |
| **전개** | 문제의 원인을 하나씩 파헤치는 탐정 과정 | 분석적 사고 모델링 |
| **절정** | Before/After/Why — 핵심 발견과 해결 | 학습의 핵심 전달 |
| **결말** | 팀에 적용할 수 있는 액션 아이템 + 토론 질문 | 실천 유도, 사회적 구성주의 |

#### 강화 원칙

| 원칙 | 적용 |
|------|------|
| **스토리텔링** (내러티브) | 4막 구조로 읽는 재미와 학습 효과를 동시에 |
| **사회적 구성주의** (Vygotsky) | 댓글/피드백 유도 질문 배치 |
| **정교화 질문** (Dunlosky 2013) | "이 패턴을 우리 프로젝트의 다른 모듈에 적용한다면?" 형태로 확장 |

---

### 4. slides (Marp 프레젠테이션)

#### 생성 방법

Marp 마크다운 형식으로 `Write` 도구를 사용하여 직접 생성한다.

#### 저장 경로

```
docs/yoda/YYYY-MM-DD-[slug]-slides.md
```

#### 15장 슬라이드 구조

| 번호 | 슬라이드 | 내용 |
|------|---------|------|
| 1 | 타이틀 | 주제 + 한 줄 요약 + 날짜 |
| 2 | 오늘의 질문 | 호기심 트리거 — 발표 전체를 관통하는 핵심 질문 |
| 3 | 현재 상황 | Before 코드 (문제 상황 시각화) |
| 4 | 어디가 아픈가 | 문제 지점 하이라이트 (⚠️ 마커) |
| 5 | Why #1 | 첫 번째 원인 분석 |
| 6 | Why #2 | 두 번째 원인 분석 (해당 시) |
| 7 | 실제 영향 | 크래시율, 성능 수치, 유지보수 비용 등 데이터 |
| 8 | 전환점 | "그러면 어떻게?" — 해결 방향 제시 |
| 9 | After 코드 | 수정된 코드 (핵심 변경만) |
| 10 | 비교 | Before vs After 나란히 배치 |
| 11 | 원칙 | 적용된 설계 원칙 명명 + 핵심 설명 |
| 12 | 시각화 | Mermaid 다이어그램 (아키텍처 변경 시) |
| 13 | Praise | 잘된 부분 칭찬 + 유지해야 할 패턴 |
| 14 | 액션 아이템 | 팀에서 즉시 적용할 수 있는 3가지 |
| 15 | 토론 | 메타인지 프롬프트 — 2개 질문으로 마무리 |

#### HTML 변환 명령

```bash
npx @marp-team/marp-cli docs/yoda/YYYY-MM-DD-[slug]-slides.md --html --output docs/yoda/YYYY-MM-DD-[slug]-slides.html
```

#### 강화 원칙

| 원칙 | 적용 |
|------|------|
| **신호 강화** | 슬라이드 제목에 핵심 메시지 집약. 볼드, 이모지로 시각적 강조 극대화 |
| **일관성 극대화** | 한 슬라이드 = 한 메시지. 불필요한 텍스트/장식 완전 제거 |
| **인지적 도제 모델링** (Collins et al.) | 전문가 사고 과정을 단계별로 노출 |

---

### 5. pptx (PowerPoint)

#### 생성 방법

`pptx` 스킬이 설치되어 있으면 위임하여 생성한다. `pptx` 스킬이 없으면 Marp 슬라이드(slides 포맷과 동일) + 별도 `*-speaker-notes.md` 파일로 발표자 노트를 생성한다.

#### 슬라이드 구성

slides 포맷의 15장 구조를 동일하게 적용한다.

#### 발표자 노트

각 슬라이드에 **발표자 노트**를 포함한다. Mayer의 양식 원칙(Modality Principle) 적용:

| 슬라이드 본문 | 발표자 노트 |
|--------------|------------|
| 키워드, 다이어그램, 코드 중심 | 상세 설명, 전환 멘트, 보충 설명 |
| 텍스트 최소화 (한 슬라이드 3줄 이하) | 발표자가 구두로 전달할 내용 전체 |
| 청중이 **보는** 것 | 발표자가 **말하는** 것 |

#### 강화 원칙

| 원칙 | 적용 |
|------|------|
| slides 원칙 전체 | 신호 강화, 일관성, 인지적 도제 모델링 동일 적용 |
| **양식 원칙** (Mayer) | 발표자 노트에 상세 배치. 슬라이드 본문은 키워드/다이어그램 중심 |

## ZPD 보정

Vygotsky의 근접 발달 영역(ZPD) 이론에 따라 `--audience` 수준별로 콘텐츠를 보정한다.

| audience | Before/After/Why 조정 | 용어 처리 | 추가 콘텐츠 |
|----------|----------------------|----------|------------|
| **junior** | Why를 상세히 설명. 기초 개념부터 단계적으로 풀어씀. After 코드에 주석을 풍부하게 배치 | 용어 풀이 포함. 약어 사용 시 풀네임 병기 | 공식 문서 링크, 레퍼런스 참조 |
| **mid** | 표준 Before/After/Why 트리플 적용. 원칙 명명 + 핵심 영향 설명 | 팀 공용 어휘 사용 | 원칙 이름만 명명 |
| **senior** | Why를 **질문으로 전환** (답을 주지 않고 사고 유도) | 축약/전문 용어 자유 사용. 학술 레퍼런스 직접 인용 가능 | 트레이드오프 분석, 대안 접근법 비교 |

## --from-review 파이프라인

review 모드의 결과를 share 모드로 변환하는 5단계 파이프라인.

### 단계 1: review 출력 탐색

현재 대화에서 가장 최근의 `/yoda review` 출력을 찾는다.

- 대화에 review 출력이 **없으면**: 에러 메시지 반환.
- 여러 review가 **있으면**: 가장 최근 것을 사용.

### 단계 2: Layer 2 + Layer 3 추출

review 출력에서 다음 요소를 추출한다:

| 추출 대상 | 출처 |
|----------|------|
| 발견 사항 (Before/After/Why) | Layer 2 |
| Praise | Layer 3 |
| Insight | Layer 3 |
| Mermaid 다이어그램 | Layer 3 (있는 경우) |
| 메타인지 프롬프트 | Layer 3 |
| 심각도 카운트 | Layer 1 |

### 단계 3: 선택 포맷에 맞게 재구성

추출한 콘텐츠를 `--format`에 지정된 포맷의 구조로 재구성한다.

### 단계 4: 포맷별 강화 원칙 적용

각 포맷의 강화 원칙(이 문서의 "포맷별 생성 전략" 섹션 참조)을 적용한다.

### 단계 5: 해당 포맷 방식으로 출력

| 포맷 | 출력 방법 |
|------|----------|
| md | `Write`로 파일 생성 → 경로 반환 |
| web | 인터랙티브 HTML 생성 → 파일 경로 반환 |
| wiki | 마크다운/위키 형식으로 생성 → 파일 경로 반환 |
| slides | `Write`로 Marp 마크다운 생성 → 변환 명령 안내 → 경로 반환 |
| pptx | `pptx` 스킬 호출 → 파일 경로 반환. 스킬 미설치 시 Marp 슬라이드 + 발표자 노트 마크다운 생성 |
